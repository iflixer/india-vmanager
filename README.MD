# Videomanager

Сервис, который управляет процессом конвертации видосов

зависимости:
- mysql



1) берем из таблицы flix_video_queue (isNull(converter) порядок - priority)
2) лочим поле converter своим хостнеймом и проверяем залочились ли
3) берем из flix_video_format информацию о способе конвертации. если нет - обновляем ошибкой format_not_found и оставляем лок
4) меняем статус на progress и updatedAt
5) вытаскиваем файл из хранишища
6) конвертим с временным именем в отдельную временну папку (по мере конверта обновляем progress и updatedAt)
7) копируем результат в хранилище по пути оригинала в папку с именем из формата + _
8) переименовываем папку без _
9) меняем статус на done и updatedAt, convert_sec=заняло времени
10) создаем запись в таблице flix_video
11) создаем запись в таблице flix_video_log


весь процесс:
1) аплоад разрешен только при редактировани статьи но не присоздании (еще нет post_id)
2) аплоадим в сервис videouploader (с post_id) который после получения последнего чанка и еще до загрузки в хранилище возвращает video_id
3) в качестве видоса в статье используем video_id, можно сохранять
4) videouploader загружает в хранилище и создает задачи на конвертации
5) videoconverter принимает задачи, конвертит и созраняет результат в бд
